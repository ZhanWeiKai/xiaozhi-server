# 添加 LLM/TTS 性能耗时日志计划

## 目标

在 xiaozhi-server 的关键节点添加计时日志，精准定位从用户消息到语音播放的耗时分布：
- **LLM 首字延迟**：API 发出到第一个 token 返回
- **LLM 流式速度**：逐 token 输出的过程，每个 token 间隔多少秒
- **TTS 耗时**：text_to_speak() 调用耗时

## 已完成的修改（第一轮）

connection.py 和 base.py 已添加基础 PERF 日志，验证结果：
- 预处理耗时 0.00s（不是瓶颈）
- LLM 首字 0.74s（不是瓶颈）
- **LLM 首字到第一句 11s（真正的瓶颈，是 LLM 流式输出每个 token 太慢）**
- TTS 0.26s（不是瓶颈）

## 本轮新增修改

### 修改文件：`core/connection.py`（仅此 1 个文件，2 处新增）

> 已有的 `_chat_start_time`（第 845 行）和 `_llm_first`（第 985 行）继续复用，无需改动。

### 新增 1：记录每个 LLM token 的输出时间

**位置**：第 1019 行，`for response in llm_responses:` 循环内，`content_arguments += content` 之后

**原代码**（function_call 路径）：
```python
if content is not None and len(content) > 0:
    content_arguments += content
```

**在其后添加**：
```python
if content is not None and len(content) > 0:
    content_arguments += content
    # === PERF: 记录每个token输出 ===
    _llm_token_time = _time_mod.time()
    if not '_llm_last_token_time' in dir():
        _llm_last_token_time = _llm_token_time
    _token_delta = _llm_token_time - _llm_last_token_time
    _llm_last_token_time = _llm_token_time
    self.logger.bind(tag=TAG).info(f"[PERF] LLM token: \"{content}\" 间隔: {_token_delta:.2f}s  累计: {_llm_token_time-_chat_start_time:.2f}s")
    # === PERF END ===
```

**说明**：
- 每次 LLM 流式返回一个 chunk（可能是一个字或几个字），立即打印时间和内容
- `间隔`：距离上一个 token 的时间差（流式速度的直观体现）
- `累计`：从 chat 开始的总耗时
- 不影响任何原有逻辑，纯打印

### 新增 2：记录 LLM 流式结束时间

**位置**：`for response in llm_responses:` 循环结束后（现有 PERF 日志之后），`except` 之前

**添加**：
```python
# for 循环结束后添加
self.logger.bind(tag=TAG).info(f"[PERF] LLM 流式输出结束，总耗时: {_time_mod.time()-_chat_start_time:.2f}s  累计token数: {len(content_arguments)}")
```

**说明**：记录整个 LLM 流式输出的总耗时和 token 总数。

用户发送 "哈哈你是谁"，假设 LLM 流式输出慢：

```
16:41:54 大模型收到用户消息: 哈哈你是谁
16:41:54 [PERF] 预处理耗时: 0.00s

16:41:56 [PERF] LLM token: "系" 间隔: 1.20s  累计: 1.94s
16:41:58 [PERF] LLM token: "Lily" 间隔: 1.80s  累计: 3.74s
16:42:01 [PERF] LLM token: "，" 间隔: 3.10s  累计: 6.84s        ← 遇到逗号，TTS开始工作
16:42:06 [PERF] LLM token: "白天" 间隔: 4.50s  累计: 11.34s
```

从 `间隔` 列就能清楚看到每个 token 之间的延迟，直观反映 LLM 流式速度。
