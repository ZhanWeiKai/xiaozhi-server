# 服务器代码修改更新流程

> 核心原则：**不要挂载整个 core/ 目录**，只挂载修改过的文件，避免覆盖容器内其他代码。

## 前置条件

- 服务器 SSH：`axonex@100.69.157.38`，sudo 密码 `Aimo123456`
- 容器名：`xiaozhi-esp32-server`
- 容器内代码路径：`/opt/xiaozhi-esp32-server/core/`
- 服务器挂载目录：`~/xiaozhi-dev-4-20/`
- docker-compose 文件：`~/xiaozhi-dev-4-20/docker-compose.yml`
- 本地工作目录：`C:\claude-project\xiaozhi-webui\xiaozhi-esp32-server\server-code`

## 步骤

### 1. 从容器提取原始文件到本地

```bash
# 从容器 copy 到服务器 /tmp
ssh axonex@100.69.157.38 "docker cp xiaozhi-esp32-server:/opt/xiaozhi-esp32-server/core/connection.py /tmp/connection_v092.py"

# 从服务器下载到本地（保持原始目录结构）
scp axonex@100.69.157.38:/tmp/connection_v092.py "C:\...\server-code\core\connection.py"
```

**注意**：本地 `server-code/` 目录要保持和容器内一致的目录结构，方便后续上传。

### 2. 在本地修改代码

在本地 IDE 中修改 `server-code/` 下的文件。

修改完成后**务必验证语法**：
```bash
cd C:\...\xiaozhi-esp32-server
python -c "import py_compile; py_compile.compile('server-code/core/connection.py', doraise=True); print('OK')"
```

### 3. 上传文件到服务器挂载目录

```bash
# 上传到 /tmp（避免权限问题）
scp "C:\...\server-code\core\connection.py" axonex@100.69.157.38:/tmp/connection.py

# 在服务器上复制到挂载目录（需要 sudo 创建目录）
ssh axonex@100.69.157.38
  sudo mkdir -p ~/xiaozhi-dev-4-20/core/providers/tts
  sudo cp /tmp/connection.py ~/xiaozhi-dev-4-20/core/connection.py
  sudo chown axonex:axonex ~/xiaozhi-dev-4-20/core/connection.py
```

### 4. 配置 docker-compose.yml 挂载（仅挂载修改的文件）

在 `volumes` 部分添加**单文件挂载**，不要挂载整个目录：

```yaml
volumes:
  - ./data:/opt/xiaozhi-esp32-server/data
  - ./models/SenseVoiceSmall/model.pt:/opt/xiaozhi-esp32-server/models/SenseVoiceSmall/model.pt
  - ./uploadfile:/opt/xiaozhi-esp32-server/uploadfile
  # 只挂载修改过的文件
  - ./core/connection.py:/opt/xiaozhi-esp32-server/core/connection.py
  - ./core/providers/tts/base.py:/opt/xiaozhi-esp32-server/core/providers/tts/base.py
```

**关键区别**：
- `./core:/opt/xiaozhi-esp32-server/core` — 整个目录挂载，**会覆盖**容器内所有 core 文件（只有你上传的文件，其余全丢失）
- `./core/connection.py:/opt/.../connection.py` — **单文件挂载**，只覆盖这一个文件，其余容器代码不受影响

### 5. 清除 pycache 并重启 docker

```bash
# 清除 pycache（防止旧字节码缓存）
echo 'Aimo123456' | sudo -S find ~/xiaozhi-dev-4-20/core -type d -name '__pycache__' -exec rm -rf {} + 2>/dev/null

# 重建容器（docker compose up -d 会重新创建容器读取挂载）
cd ~/xiaozhi-dev-4-20
docker compose up -d xiaozhi-esp32-server
```

**注意**：`docker compose restart` 不会重新读取 compose 文件的挂载配置，必须用 `docker compose up -d`。

### 6. 验证

```bash
# 确认容器正常启动
docker logs xiaozhi-esp32-server --tail 5

# 确认挂载文件生效（在容器内检查）
docker exec xiaozhi-esp32-server grep -c 'PERF' /opt/xiaozhi-esp32-server/core/connection.py
```

## 回滚

去掉挂载即可恢复容器原始代码：

```bash
# 删除挂载行后重启
cd ~/xiaozhi-dev-4-20
docker compose up -d xiaozhi-esp32-server
```

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError: No module named 'core.xxx'` | 挂载了整个 core/ 目录，但只有部分文件 | 改为单文件挂载 |
| 改了代码但不生效 | pycache 缓存了旧字节码 | `sudo find ... -name '__pycache__' -exec rm -rf {} +` |
| docker compose restart 后挂载不生效 | restart 不会重新读取 compose 文件 | 用 `docker compose up -d` |
| `Permission denied` 上传文件 | core 目录由 root 创建 | `sudo cp` + `sudo chown` |
| 语法错误导致容器启动失败 | 本地修改有语法问题 | 先在本地 `python -c "import py_compile; ..."` 验证 |
